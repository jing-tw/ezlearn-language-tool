8/12/2005
a. [滑鼠操作] 滑鼠可以在視窗的任何地方, 按左鍵拖曳視窗移動. (為 skin 作準備 => 如果有 skin, 則視窗上的標題則必須取消, 以維持美觀)
   作法:
        利用程式模擬滑鼠拖曳的動作
        
   // ====================================== 關鍵程式片段 ====================================
   // [滑鼠功能] 當滑鼠移動時
		bool DRAGGING=	true;
		bool m_dwFlags=false;
		int Py,Px;
		private void AVPlayer_MouseMove(object sender, System.Windows.Forms.MouseEventArgs e) {
			if ( m_dwFlags && DRAGGING ) {
				this.Left+=e.X-Px;
				this.Top+=e.Y-Py;
			}
		}

		//[滑鼠功能] 當滑鼠左鍵彈起時
		private void AVPlayer_MouseUp(object sender, System.Windows.Forms.MouseEventArgs e) {
			switch (e.Button) {
				case MouseButtons.Left:
					if ( m_dwFlags && DRAGGING ) {
	                               // 若目前是 dragging 狀態
						// m_dwFlags &= ~DRAGGING;
						m_dwFlags=!DRAGGING;
					}
					break;
			}
		}

		
		// [滑鼠功能] 當滑鼠左鍵按下時
		private void AVPlayer_MouseDown(object sender, System.Windows.Forms.MouseEventArgs e) {
			switch (e.Button) {
				case MouseButtons.Left:
					// if ( !(m_dwFlags & DRAGGING) ) {
					if ( !(m_dwFlags && DRAGGING) ){
	                               // 若目前不是 dragging 狀態
						Py=e.Y; Px=e.X;

						// m_dwFlags |= DRAGGING;
						m_dwFlags=DRAGGING;
					}
					break;
			}

		}
		
   // ====================================== end of 關鍵程式片段 ====================================

[Bug 修正]
1.有時會出現無聲的情況問題 (jungping 提出)
原因: 主要是可能是因為陽春的 Microsoft.DirectX.AudioVideoPlayback 組件啟動時間很慢, 導致有時會無聲. 
      這種現象在速度較慢的機器上, 會更明顯. 最終解決方案是重新利用 C++ 使用 DirectShow SDK 寫一個
      更有彈性的播放器, 藉以模擬 AudiovideoPlayback. 但是這個部分可能要留到後面再處理.
      
      目前的解決方案是根據 AudioVideoPlayback 的特性, 直接將播放完成後的動作. 
      由 停止播放 改成 持續播放 一個無聲片段, 來改進啟動時間慢的問題, 因為這樣的動作將會使 AudiovideoPlayback
      沒有關閉與啟動的動作. 看看是否能用現有的資源改善目前遭遇的問題. 進而達到最大的開發效益.
      
      基本上, 經過我修改後. 我的機器(1.4 GHz) 已經沒有發現無聲的問題了. 但是不知道其他網友是否還會遭遇這個問題?
      
狀態: 持續解決中
